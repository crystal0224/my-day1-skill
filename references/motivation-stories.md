# 동기부여 스토리

> 각 필수 미션 Phase A 시작 시, 사용자 프로파일의 직업에 맞는 스토리를 선택하여 출력한다.
> 구조: 공감 질문 → 실제 사례 (Before/After) → 이 미션 완료 후 얻는 것
>
> 필수 7개 미션 x 9개 직업 = 63개 스토리

---

## 미션 0: Setup

> **미션 요약**: Claude Code 설치 및 첫 실행

### HR/인사 담당자

채용 시즌마다 공고 올리고, 이력서 정리하고, 면접 일정 잡느라 정신없으시죠?

**실제 사례:**
김OO님(HR 3년차)은 Claude Code 설치 후, 첫 대화에서 "우리 회사 채용 프로세스 정리해줘"라고 했더니 기존 문서를 기반으로 5분 만에 깔끔한 프로세스 문서가 나왔습니다. 예전에는 Word에서 2시간 걸리던 일이었습니다.

**이 미션을 완료하면:**
- Claude Code가 내 컴퓨터에 설치됩니다
- 첫 대화로 AI 업무 도우미를 만나게 됩니다
- 다음 미션부터 실무에 바로 쓸 수 있는 기능을 배웁니다

---

### 백엔드/프론트엔드/풀스택 개발자

새 프로젝트마다 보일러플레이트 세팅하고, lint 설정 복사하고, CI/CD 파이프라인 만드느라 반나절 날리는 거 지겹지 않으세요?

**실제 사례:**
이OO님(백엔드 5년차)은 Claude Code를 설치하자마자 "Express + TypeScript + Prisma 프로젝트 세팅해줘"라고 했습니다. 3분 만에 폴더 구조, 설정 파일, 기본 라우트까지 완성되었습니다. 이전에는 공식 문서 보면서 40분 걸리던 작업이었습니다.

**이 미션을 완료하면:**
- 터미널에서 바로 Claude Code를 실행할 수 있습니다
- 코딩 파트너가 터미널 안에 상주하게 됩니다
- 개발 환경 세팅부터 코드 리뷰까지 한곳에서 처리됩니다

### UI/UX 디자이너

개발팀에 디자인 의도를 전달할 때 "여기 패딩 16px이고, 컬러는 #3B82F6이고..." 일일이 적는 게 귀찮지 않으세요?

**실제 사례:**
박OO님(UX 디자이너)은 Claude Code를 설치한 뒤 "Figma 디자인 토큰을 CSS 변수로 바꿔줘"라고 했더니, 디자인 시스템 전체가 코드로 변환되었습니다. 개발자와의 소통 시간이 절반으로 줄었습니다.

**이 미션을 완료하면:**
- 디자인 의도를 코드로 직접 전달할 수 있는 도구가 생깁니다
- 개발자에게 "이거 좀 고쳐주세요" 대신 직접 수정도 가능해집니다
- 디자인 시스템 관리가 훨씬 체계적으로 됩니다

### 제품 기획자/PM

스프린트 리뷰 자료 만들 때 Jira, Confluence, Slack 돌아다니면서 정보 긁어모으느라 금요일 오후가 사라지지 않나요?

**실제 사례:**
최OO님(PM 4년차)은 Claude Code 첫 실행 후 "이번 스프린트 완료된 이슈 정리해줘"라고 했더니, git 로그에서 자동으로 변경사항을 추출해 리뷰 문서를 만들어줬습니다. 매주 2시간 걸리던 작업이 10분으로 줄었습니다.

**이 미션을 완료하면:**
- 데이터 기반 의사결정을 도와줄 AI 어시스턴트가 생깁니다
- 반복적인 문서 작업을 자동화할 준비가 됩니다
- 기획에 더 많은 시간을 쓸 수 있게 됩니다

### 데이터 분석가

분석할 때마다 "이 컬럼 의미가 뭐였지?" DB 스키마 문서 찾느라 시간 쓰는 거 아깝지 않나요?

**실제 사례:**
정OO님(데이터 분석가)은 Claude Code 설치 후 "이 CSV 파일에서 월별 매출 트렌드 분석해줘"라고 했더니, 자동으로 pandas 코드를 작성하고 실행까지 해서 그래프와 인사이트를 뽑아줬습니다. 직접 코드 짤 때보다 3배 빨랐습니다.

**이 미션을 완료하면:**
- SQL 쿼리, Python 스크립트를 말로 시킬 수 있게 됩니다
- 데이터 정제부터 시각화까지 한번에 처리됩니다
- 분석 자체에 더 집중할 수 있게 됩니다

### 마케터/콘텐츠 제작자

캠페인 보고서 만들 때 Google Analytics, 광고 플랫폼, CRM 데이터 따로따로 복사해서 합치는 거 번거롭지 않나요?

**실제 사례:**
강OO님(퍼포먼스 마케터)은 Claude Code를 설치한 첫날, "지난달 광고 성과 데이터를 표로 정리해줘"라고 했더니 CSV 파일을 분석해서 채널별 ROAS, CPC, 전환율을 깔끔한 표로 만들어줬습니다. PPT에 복사하기만 하면 끝이었습니다.

**이 미션을 완료하면:**
- 콘텐츠 초안 작성부터 데이터 정리까지 AI가 도와줍니다
- 반복적인 포맷팅 작업에서 해방됩니다
- 크리에이티브한 일에 더 집중할 수 있게 됩니다

### 일반 사무/관리

매달 같은 양식의 보고서를 만들면서 "이거 작년에 어떻게 했더라?" 메일함 뒤지는 거 지치지 않았나요?

**실제 사례:**
한OO님(총무 담당)은 Claude Code 설치 후 "지난달 경비 내역 정리해줘"라고 했더니, 엑셀 파일을 읽고 부서별로 자동 분류해서 보고서를 만들어줬습니다. 손으로 하면 반나절이 걸리던 일이 15분 만에 끝났습니다.

**이 미션을 완료하면:**
- 반복 업무를 도와줄 AI 어시스턴트가 생깁니다
- 양식 작성, 데이터 정리 같은 단순 업무가 빨라집니다
- 더 중요한 업무에 시간을 쓸 수 있게 됩니다

### 연구원/학생

논문 읽을 때 "이 논문이 어떤 방법론을 썼더라?" 찾느라 PDF를 왔다 갔다 하는 거 비효율적이지 않나요?

**실제 사례:**
조OO님(대학원 석사과정)은 Claude Code를 설치하고 "이 논문 PDF 요약해줘"라고 했더니 연구 목적, 방법론, 주요 발견, 한계점을 구조화해서 정리해줬습니다. 논문 하나 읽는 시간이 1시간에서 15분으로 줄었습니다.

**이 미션을 완료하면:**
- 논문 읽기, 코드 작성, 데이터 분석을 도와줄 AI가 생깁니다
- 연구 아이디어를 빠르게 프로토타이핑할 수 있게 됩니다
- 연구 생산성이 확 올라갑니다

### 대학생/취준생

포트폴리오 만들고 싶은데 코딩은 모르고, 노코드 툴은 한계가 있고, 외주 맡기자니 비용이 부담되지 않나요?

**실제 사례:**
윤OO님(컴공 3학년)은 Claude Code를 설치한 첫날, "내 GitHub 프로필 README를 멋지게 만들어줘"라고 했더니 기술 스택 배지, 프로젝트 하이라이트, 통계 위젯까지 포함된 프로필이 완성되었습니다. 다른 친구들이 "어떻게 만들었어?" 하고 물어볼 정도였습니다.

**이 미션을 완료하면:**
- 코딩 학습과 프로젝트를 가속시킬 AI 멘토가 생깁니다
- 포트폴리오, 과제, 사이드 프로젝트를 훨씬 빠르게 만들 수 있습니다
- 취업 준비에서 남들보다 한발 앞설 수 있습니다

## 미션 1: Experience

> **미션 요약**: 7일 후 모습을 미리 체험 (Skill 실행, AskUserQuestion, Claude에게 물어보기)

### HR/인사 담당자

"AI가 진짜 업무에 도움이 돼?" 반신반의하면서 시작하셨죠?

**실제 사례:**
김OO님(HR)은 체험에서 "우리 팀 업무를 개선해줘"라고 모호하게 요청했을 때와, AskUserQuestion을 활용해 "어떤 팀?", "가장 큰 병목은?" 질문받으며 진행했을 때의 차이를 보고 놀랐습니다. AI가 알아서 할 줄 알았는데, 제대로 시키는 법이 따로 있다는 걸 알게 되었습니다.

**이 미션을 완료하면:**
- AI에게 효과적으로 요청하는 방법을 알게 됩니다
- "AI가 알아서 해주겠지" → "이렇게 시키면 정확히 해준다"로 바뀝니다
- 한 마디로 복잡한 업무를 자동화하는 미래를 직접 봅니다

---

### 백엔드/프론트엔드/풀스택 개발자

"또 AI 툴이야? GitHub Copilot이랑 뭐가 다른데?" 생각하고 계시죠?

**실제 사례:**
이OO님(풀스택 개발자)은 체험에서 Claude Code가 단순 코드 완성이 아니라, git log를 분석하고, 파일을 생성하고, 테스트를 실행하는 등 터미널 전체를 제어하는 것을 보고 "이건 Copilot과 차원이 다르다"고 했습니다. 코드 에디터 안의 자동완성이 아니라 터미널 안의 개발 파트너였습니다.

**이 미션을 완료하면:**
- Claude Code가 기존 AI 코딩 도구와 어떻게 다른지 직접 체험합니다
- Skill 실행으로 복잡한 워크플로우가 한 줄로 가능하다는 걸 봅니다
- "모르면 Claude에게 물어보면 된다"는 핵심 패턴을 익힙니다

### UI/UX 디자이너

"개발 도구인데 디자이너인 내가 왜 배워야 해?" 생각하고 있지 않나요?

**실제 사례:**
박OO님(UI 디자이너)은 체험에서 Claude Code에게 "이 앱의 사용성을 개선해줘. 모호한 부분은 질문으로 확인해"라고 했더니, "타겟 사용자는 누구인가요?", "가장 많이 사용하는 기능은?" 등 UX 리서치에서 묻는 질문을 해서 놀랐습니다. 디자이너의 사고방식으로 대화할 수 있는 도구였습니다.

**이 미션을 완료하면:**
- Claude Code가 디자인 업무에도 활용 가능하다는 걸 직접 확인합니다
- "코딩 도구" → "업무 자동화 도구"로 인식이 바뀝니다
- 디자인 시스템 관리, 프로토타이핑에 활용할 가능성을 봅니다

### 제품 기획자/PM

"기획자가 코딩 도구를 왜 써야 하지?" 아직 확신이 안 드시죠?

**실제 사례:**
최OO님(PM)은 체험에서 "우리 팀 업무를 개선해줘"라고 막연하게 요청했다가, AskUserQuestion 패턴을 쓰자 Claude가 "팀 규모는?", "주요 병목은?", "현재 사용 중인 도구는?" 순서로 물어왔습니다. "이거 내가 맨날 하는 요구사항 정리 프로세스잖아!" 하며 바로 활용법이 떠올랐습니다.

**이 미션을 완료하면:**
- 요구사항 명확화를 AI가 도와주는 경험을 합니다
- PRD 작성, 스프린트 정리 등에 바로 적용할 아이디어가 생깁니다
- "기획자한테 최적화된 도구"라는 확신이 생깁니다

### 데이터 분석가

정OO님(데이터 분석가)은 체험에서 "이 데이터 분석해줘"라고 했을 때와, "매출 데이터인데, 월별 트렌드랑 이상치 탐지를 해줘. 모호한 부분은 질문해"라고 했을 때의 결과 차이를 보고 "프롬프트 품질이 분석 품질을 결정한다"는 걸 깨달았습니다.

**이 미션을 완료하면:**
- Claude Code가 Jupyter와 다른 방식으로 분석을 가속시키는 걸 봅니다
- 파일 읽기, 코드 작성, 실행을 한번에 시킬 수 있다는 걸 알게 됩니다

- ChatGPT와 Claude Code의 근본적인 차이를 이해합니다

**이 미션을 완료하면:**
- AI가 단순 대화를 넘어서 실제 작업을 수행하는 걸 직접 봅니다
- 반복 업무 자동화의 가능성을 구체적으로 느낍니다
