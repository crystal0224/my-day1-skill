# My Day 1: 개인화 온보딩

사용자 프로파일을 기반으로 맞춤형 예시와 설명을 제공하는 Claude Code 온보딩 스킬입니다.

이 스킬이 호출되면 아래 **STOP PROTOCOL**을 반드시 따른다.

---

## 프로파일링 (최초 1회)

스킬 시작 시 자동으로 프로파일을 확인하고 필요 시 수집합니다.

### 1. 프로파일 확인

```
Read ~/.claude/memory/user-profile.md
```

- **파일 존재**: 프로파일 로드 → 확인 질문 (아래 2단계)
- **파일 없음**: 프로파일링 시작 (아래 3단계)

### 2. 프로파일 확인 (파일 존재 시)

프로파일을 읽어서 사용자에게 표시:

```
현재 저장된 프로파일:
- 직업: [직업]
- 전공: [전공]
- 터미널 경험: [경험]
```

AskUserQuestion으로 확인:

```json
{
  "questions": [{
    "question": "저장된 프로파일로 진행할까요?",
    "header": "프로파일 확인",
    "options": [
      {"label": "네, 이대로 진행합니다"},
      {"label": "아니요, 프로파일을 다시 설정하겠습니다"}
    ],
    "multiSelect": false
  }]
}
```

- "네, 이대로 진행합니다" → 블록 선택으로 이동 (아래 5단계)
- "아니요, 프로파일을 다시 설정하겠습니다" → 프로파일링 진행 (아래 3단계)

## STOP PROTOCOL — 절대 위반 금지

> 이 프로토콜은 이 스킬의 최우선 규칙이다.
> 아래 규칙을 위반하면 수업이 망가진다.

### 각 블록은 반드시 2턴에 걸쳐 진행한다

```
┌─ Phase A (첫 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 블록 파일의 EXPLAIN 섹션을 읽는다    │
│ 2. 기능을 설명한다                                        │
│ 3. references/에서 해당 블록 파일의 EXECUTE 섹션을 읽는다    │
│ 4. "지금 직접 실행해보세요"라고 안내한다                     │
│ 5. ⛔ 여기서 반드시 STOP. 턴을 종료한다.                    │
│                                                          │
│ ❌ 절대 하지 않는 것: 퀴즈 출제, QUIZ 섹션 읽기             │
│ ❌ 절대 하지 않는 것: AskUserQuestion 호출                  │
│    ⚠️ 예외: 스킬 시작 시 프로파일링은 허용                  │
│ ❌ 절대 하지 않는 것: "실행해봤나요?" 질문                   │
└──────────────────────────────────────────────────────────┘

  ⬇️ 사용자가 돌아와서 "했어", "완료", "다음" 등을 입력한다

┌─ Phase B (두 번째 턴) ──────────────────────────────┐
│ 1. references/에서 해당 블록 파일의 QUIZ 섹션을 읽는다       │
│ 2. AskUserQuestion으로 퀴즈를 출제한다                     │
│ 3. 정답/오답 피드백을 준다                                 │
│ 4. 다음 블록으로 이동할지 AskUserQuestion으로 묻는다         │
│ 5. ⛔ 다음 블록을 시작하면 다시 Phase A부터.                │
└──────────────────────────────────────────────────────────┘
```

### 핵심 금지 사항 (절대 위반 금지)

1. **Phase A에서 AskUserQuestion을 호출하지 않는다** — 설명 + 실행 안내 후 바로 Stop
   - ⚠️ 예외: 스킬 시작 시 프로파일링(프로파일 확인/수집)은 허용
2. **Phase A에서 퀴즈를 내지 않는다** — QUIZ 섹션은 Phase B에서만 읽는다
3. **Phase A에서 "실행해봤나요?"를 묻지 않는다** — 사용자가 먼저 말할 때까지 기다린다
4. **한 턴에 EXPLAIN + QUIZ를 동시에 하지 않는다** — 반드시 2턴으로 나눈다

### 공식 문서 URL 출력 (절대 누락 금지)

모든 블록의 Phase A 시작 시, 해당 reference 파일 상단의 `> 공식 문서:` URL을 **반드시 그대로 출력**한다.

```
📖 공식 문서: [URL]
```

- reference 파일에 URL이 여러 개 있으면 전부 출력한다
- URL을 요약하거나 생략하지 않는다
- 참가자가 직접 클릭해서 공식 문서를 볼 수 있어야 한다

### Phase A 종료 시 필수 문구

Phase A의 마지막에는 반드시 아래 형태의 문구를 출력하고 Stop한다:

```
---
👆 위 내용을 직접 실행해보세요.
실행이 끝나면 "완료" 또는 "다음"이라고 입력해주세요.
```

이 문구 이후에 어떤 도구 호출(AskUserQuestion 포함)이나 추가 텍스트도 출력하지 않는다.

5. **EXECUTE 섹션 — 맞춤 예시 추가**

   원본 명령어(`/init`, `/memory`)는 그대로 유지하고, 각 명령어 설명에 프로파일 맞춤 예시를 추가한다.

   **터미널 경험이 "처음이에요"인 경우 — EXECUTE 맨 앞에 추가:**
   ```markdown
   ### 터미널 여는 방법 (처음이라면 여기부터!)
   - Mac: Spotlight (Cmd + Space) → "터미널" 입력 → Enter
   - Windows: 시작 메뉴 → "cmd" 검색 → Enter
   - 터미널이 열리면 까만 화면에 커서가 깜빡입니다. 여기에 명령어를 입력하면 됩니다.
   ```

   **터미널 경험이 "해본 적 있어요"인 경우:**
   - 터미널 여는 법은 생략
   - 명령어 설명은 간결하게

   **터미널 경험이 "익숙해요"인 경우:**
   - 터미널 여는 법 생략
   - 명령어 설명도 간소화
