# Essential Path 테스트 시나리오

## 목표

비개발자가 my-day1 스킬을 끝까지 완료하도록 개선한 Essential Path 기능들을 검증합니다.

## 성공 기준

- ✅ 필수 7개 미션 완료율 **80% 이상**
- ✅ 중도 이탈률 **20% 이하**
- ✅ "실무에 적용하겠다" 응답 **70% 이상**
- ✅ 용어 이해도 (퀴즈 정답률) **80% 이상**

---

## 시나리오 1: 터미널 처음 사용하는 HR

**프로파일:**
- 직업: HR/인사 담당자
- 터미널 경험: 처음이에요

**학습 경로:** 빠른 시작 (필수 7개)

### 검증 포인트

#### 1. 용어 비유가 이해되는가?

**테스트:**
```
/my-day1 실행
→ 프로파일 설정 (HR, 처음이에요)
→ 빠른 시작 선택
→ 미션 0 Phase A 진행
```

**확인 사항:**
- [ ] CLI 용어 출현 시 "Excel 수식처럼, 텍스트로 컴퓨터에 명령하는 방법" 비유 표시
- [ ] 터미널 용어 출현 시 "컴퓨터와 대화하는 텍스트 창" 비유 표시
- [ ] git 용어 출현 시 "Word의 '변경 내용 추적' 강력 버전" 비유 표시

**예상 결과:** ✅ 모든 용어에 HR 맞춤 비유 표시

---

#### 2. 동기부여 스토리가 공감되는가?

**테스트:**
```
미션 0 Phase A 시작
```

**확인 사항:**
- [ ] "💡 왜 이 미션을 해야 하나요?" 섹션 표시
- [ ] HR 맞춤 스토리: "매번 똑같은 채용 이메일 보내는 데 지치셨나요?"
- [ ] 실제 사례: "김OO님(HR)은 Claude Code로 채용 공고 20개를 5분 만에 업데이트"
- [ ] "이 미션을 완료하면" 목표 3가지 명시

**예상 결과:** ✅ 동기부여 스토리 표시 및 공감

---

#### 3. 실습을 따라할 수 있는가?

**테스트:**
```
미션 0 Phase A 실습 중 의도적으로 막힘
→ "help" 입력
```

**확인 사항:**
- [ ] Help 메뉴 4가지 옵션 표시
- [ ] "명령어가 안 돼요" 선택 시 진단 + 해결법 제공
- [ ] "무슨 말인지 모르겠어요" 선택 시 쉬운 재설명
- [ ] "이 미션 건너뛰고 싶어요" 선택 시 필수 미션 경고 + skip 가능

**예상 결과:** ✅ Help 시스템 작동, 막힘 해결

---

#### 4. 7개 완료 후 실무 적용 의지가 생기는가?

**테스트:**
```
필수 7개 미션 모두 완료
```

**확인 사항:**
- [ ] 🎯 기초 완료 배지 (미션 0,1,3-1 완료 시)
- [ ] 🏆 핵심 완료 배지 (필수 7개 완료 시)
- [ ] "Claude Code 핵심 마스터! 실무 투입 가능!" 메시지
- [ ] 다음 단계 안내 (실무 적용 / 심화 미션 도전)

**예상 결과:** ✅ 배지 획득, 실무 적용 동기 부여

## 시나리오 2: 터미널 조금 써본 디자이너

**프로파일:**
- 직업: UI/UX 디자이너
- 터미널 경험: 해본 적 있어요

**학습 경로:** 전체 과정 (13개)

### 검증 포인트

#### 1. 동기부여 스토리가 공감되는가?

**테스트:**
```
미션 3-1 (CLAUDE.md) Phase A 시작
```

**확인 사항:**
- [ ] 디자이너 맞춤 스토리: "디자인 시스템 문서 업데이트할 때마다 '이거 어디 있더라?'"
- [ ] 실제 사례: "박OO님(디자이너)은 디자인 시스템 일관성을 100% 유지"
- [ ] 용어 비유: CLI → "Figma 단축키의 강력 버전"

**예상 결과:** ✅ 디자이너 맞춤 스토리 및 비유

---

#### 2. 중간에 이탈하지 않고 완료하는가?

**테스트:**
```
미션 5개 완료 후 중단
→ 다음 날 /my-day1 재시작
```

**확인 사항:**
- [ ] "👋 다시 오셨네요!" 재진입 메시지
- [ ] 진행 상황 요약 (완료 미션, 진행률, 배지)
- [ ] "이어서 하기 / 처음부터 다시 / 특정 미션 선택" 옵션
- [ ] 이어서 하기 선택 시 다음 미션 자동 시작

**예상 결과:** ✅ 재진입 플로우 작동, 이어서 하기 가능

---

#### 3. 심화 미션도 "왜 필요한지" 이해되는가?

**테스트:**
```
미션 3-5 (Agent Teams) Phase A 시작
```

**확인 사항:**
- [ ] 심화 미션 태그 🔷 표시
- [ ] 동기부여 스토리: "디자인 프로젝트 팀" 비유
- [ ] "이 미션을 건너뛰고 싶어요" → "심화 미션이라 건너뛰어도 됩니다" 안내

**예상 결과:** ✅ 심화 미션 동기부여 및 선택적 학습 이해

---

#### 4. 전체 완료 시 성취감이 있는가?

**테스트:**
```
13개 미션 모두 완료
```

**확인 사항:**
- [ ] 🚀 마스터 배지 획득
- [ ] "Claude Code 전문가 등극! 모든 기능을 자유자재로!" 메시지
- [ ] "🎓 온보딩 완료" 축하 메시지

**예상 결과:** ✅ 마스터 배지, 성취감 획득

## 시나리오 3: 개발 경험 있는 PM

**프로파일:**
- 직업: 제품 기획자/PM
- 터미널 경험: 익숙해요

**학습 경로:** 커스텀 (필요한 미션만)

### 검증 포인트

#### 1. 필요한 미션만 골라서 빠르게 완료 가능한가?

**테스트:**
```
/my-day1 실행
→ 커스텀 경로 선택
→ 미션 3-1, 3-2, 3-4만 선택
```

**확인 사항:**
- [ ] 미션 선택 화면에 ⭐필수/🔷심화 태그 표시
- [ ] 선택한 미션만 순서대로 진행
- [ ] 완료 후 진행 상황에 선택한 미션만 표시

**예상 결과:** ✅ 커스텀 경로 작동, 선택적 학습 가능

---

#### 2. 이미 아는 내용은 skip이 자연스러운가?

**테스트:**
```
미션 0 (Setup) Phase A 시작
→ "skip" 입력
```

**확인 사항:**
- [ ] 즉시 다음 미션으로 이동
- [ ] 필수 미션 skip 시 "나중에 꼭 하셔야 🏆 배지 획득 가능" 안내
- [ ] user-profile.md에 skip한 미션 기록 안 됨

**예상 결과:** ✅ Skip 자연스럽게 작동

---

#### 3. 경험자 수준에 맞는 설명인가?

**테스트:**
```
미션 진행 중 용어 출현
```

**확인 사항:**
- [ ] PM 맞춤 비유: CLI → "프로젝트 커맨드 센터"
- [ ] git → "프로젝트 타임머신"
- [ ] 터미널 경험 익숙함 → 기본 설명 생략 가능

**예상 결과:** ✅ 경험자 수준 맞춤 설명
